# Agentic AI Agent — Learning Repository

This repository is the central platform for all work completed during the **Agentic AI** training program. All assignments are issued via **GitHub Issues**, developed on dedicated **feature branches**, and merged through **Pull Requests** — mirroring real-world, industry-standard Git workflows.

---

## Purpose

The goal of this repository is to build practical, hands-on experience in:

- Designing and developing **Agentic AI systems** that can reason, plan, and act autonomously.
- Applying **LLM-based tooling** (prompt engineering, tool use, memory, and multi-agent orchestration).
- Following **professional software engineering practices**: branching strategies, code reviews, CI/CD, and documentation.

---

## Learning Objectives

By the end of this training, I aim to:

1. Understand the architecture and principles behind agentic AI systems.
2. Build autonomous agents capable of using tools, memory, and multi-step reasoning.
3. Design and evaluate effective prompts for large language models.
4. Develop reusable AI tools and integrate them into agent pipelines.
5. Write clean, testable, and well-documented code for AI-driven projects.
6. Collaborate using Git: feature branches, pull requests, and peer code review.

---

## Repository Structure

```
AGENTIC-AI-AGENT/
│
├── docs/               # Documentation, notes, research, and design specs
├── assignments/        # Individual assignment submissions (one folder per task)
├── projects/           # Larger project work spanning multiple assignments
├── agents/             # Agent implementations and configurations
├── tools/              # Reusable tools and utilities consumed by agents
├── prompts/            # Prompt templates, system instructions, and experiments
└── tests/              # Unit tests, integration tests, and evaluation scripts
```

---

## Workflow

1. A new assignment or task is opened as a **GitHub Issue**.
2. A **feature branch** is created from `main` (e.g., `feature/issue-3-memory-agent`).
3. Work is committed to the feature branch with clear, descriptive commit messages.
4. A **Pull Request** is opened against `main`, referencing the original issue.
5. After review and approval, the branch is merged and the issue is closed.

---

## Tech Stack

- **Language:** Python 3.10+
- **LLM Providers:** OpenAI, Anthropic (Claude), or open-source models
- **Frameworks:** LangChain, LlamaIndex, or custom agent loops
- **Version Control:** Git + GitHub
- **Testing:** pytest

## Getting Started

```bash
# Clone the repository
git clone https://github.com/<your-username>/AGENTIC-AI-AGENT.git
cd AGENTIC-AI-AGENT

# (Optional) Create and activate a virtual environment
python -m venv .venv
.venv\Scripts\activate      # Windows
# source .venv/bin/activate  # macOS / Linux

# Install dependencies (added as the project grows)
pip install -r requirements.txt
```

## Contributing

All contributions follow the **GitHub Flow**:

- Branch off `main` for every issue.
- Keep commits atomic and well-described.
- Open a PR, request a review, and address feedback before merging.
- Never push directly to `main`.
